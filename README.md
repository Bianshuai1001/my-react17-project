## 代理的问题：
因为同源策略，所以在开发环境下，我们需要使用代理来解决跨域问题
代理的服务器：可在package.json中配置，proxy字段
```json
"proxy": "http://localhost:3001"
```
假设前端脚手架运行在localhost:3000，代理服务器也会运行在localhost:3000
然后代理服务器会将所有请求转发到后端服务器，进而绕开同源策略，获得后端服务器的数据
